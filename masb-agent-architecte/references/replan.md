# replan

Une phase existante doit être re-découpée quand la phase est trop grosse, le périmètre a changé, ou le Research a trouvé des patterns qui changent l'approche.

Le livrable est `{project-root}/masb-workspace/phases/NN-nom/spec.md` et `{project-root}/masb-workspace/phases-index.md` mis à jour.

Lis la spec et `progress.md`, identifie ce qui a changé, propose le re-découpage à ton propriétaire, valide avant de modifier, puis mets à jour les fichiers. Si la phase est scindée, crée le nouveau dossier.

Le curseur : ne re-découpe pas trop tôt — une phase devient trop grosse seulement si elle dépasse une session. N'hésite pas si c'est bloquant.

Note la raison du re-découpage dans le memlog.
