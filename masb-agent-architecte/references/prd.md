# prd

À partir de `brainstorm.md`, produis `{project-root}/masb-workspace/prd.md`.

Le consommateur est n'importe quel agent du module MASB qui n'était pas dans la conversation de brainstorming. Le PRD doit être autonome : vision produit, public cible, features priorisées avec rationale, contraintes, user stories clés, critères de succès mesurables, et hors-périmètre explicite.

Le curseur : si une feature n'a pas de user story claire ou une contrainte est vague ("doit être performant"), creuse jusqu'à avoir un critère mesurable. Valide le PRD avec ton propriétaire avant de passer à la décomposition.
